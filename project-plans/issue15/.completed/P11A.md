## GhClient Wiring Note

No `GhClient` method calls (e.g. `list_issues`, `create_comment`) exist in `src/app_input/issues.rs`. The key routing layer is intentionally **pure** — it maps key events to `AppEvent` variants only. `GhClient` wiring is deferred to the side-effect / command layer that will process these events. This is consistent with the architecture: `resolve_issues_key_event()` is documented as "side-effect-free and testable without iocraft hooks." The actual `GhClient` calls will be wired when the event dispatch layer processes events like `InlineSubmit`, `IssuesEnter`, `ApplySearch`, etc. This is a planned scope boundary, not a gap.
